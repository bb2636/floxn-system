[목표]
현재 PDF 다운로드는 화면 캡쳐 기반이라 결과물이 웹 UI처럼 보입니다.
이를 전면 중단하고, “데이터 바인딩 + PDF 전용 템플릿 + PDF 병합(merge)” 방식으로
보험/복구업계 제출용 문서 형태의 단일 final.pdf를 생성하도록 구현해 주세요.

[절대 금지]
- 웹 화면 캡쳐/스크린샷 기반 PDF 생성 금지
- PDF 내에 ‘전체 다운로드’, 다운로드 아이콘, 버튼/탭 UI 등 웹 UI 요소 포함 금지
- CSS scale로 억지로 맞추는 방식 금지

--------------------------------------------
1) 다운로드 전 선택 UI(프론트 → 서버 payload)
--------------------------------------------
다운로드 전에 모달에서 사용자가 “포함할 항목”을 선택합니다.

A. 상단 섹션 선택 체크박스
- 현장입력
- 도면
- 증빙자료
- 견적서
- 기타사항

B. 증빙자료는 탭 + 파일 선택(멀티) 지원
- 탭: 전체 / 현장사진 / 기본자료 / 증빙자료 / 청구자료 (현재 시스템과 동일)
- 탭 내 파일 카드/리스트에서 체크박스로 개별 파일 선택
- 선택한 파일만 PDF에 포함되도록 해야 함
- 증빙자료 섹션 체크를 끄면 탭/파일 선택 UI 비활성화

C. 프론트에서 서버로 아래 payload 전송
(키명은 아래 구조를 유지해 주세요)

{
  "caseId": "<사건/접수 고유 ID>",
  "sections": {
    "cover": true,
    "fieldReport": true,
    "drawing": true,
    "evidence": true,
    "estimate": true,
    "etc": false
  },
  "evidence": {
    "tab": "현장사진",
    "selectedFileIds": ["file_1", "file_3", "file_9"]
  }
}

- selectedFileIds는 “실제로 PDF에 포함할 파일”만 전달
- ‘전체 선택’이면 프론트에서 해당 탭의 전체 파일 id를 selectedFileIds로 채워 전송

--------------------------------------------
2) 최종 PDF 구성 순서(단일 final.pdf)
--------------------------------------------
사용자 선택(sections)에 따라 아래 순서로 final.pdf를 구성합니다.

0p. 표지(현장출동확인서)  (sections.cover === true 일 때)
1p. 출동보고서(표 기반 정형 양식) (sections.fieldReport)
2p. 현장 피해상황 도면(정형 양식 + 도면 이미지 삽입) (sections.drawing)
3p~. 증빙자료(이미지/문서) (sections.evidence)
그 다음. (증빙자료 중 PDF 첨부는 원본 PDF 페이지를 그대로 append)
견적서(표 기반 정형 양식, 섹션 단위 페이지 분리) (sections.estimate)
기타사항(있으면 정형 양식) (sections.etc)

* 최종 결과는 “단일 PDF”로 다운로드되어야 합니다.
* UI에 체크된 섹션만 포함합니다.

--------------------------------------------
3) 생성 방식(핵심 구현 방식)
--------------------------------------------
A. HTML 템플릿 → Puppeteer/Playwright로 PDF 생성
- 표지, 출동보고서, 도면, 증빙(이미지 페이지), 견적서, 기타사항은
  모두 “PDF 전용 HTML 템플릿”으로 렌더 후 PDF로 생성합니다.
- 화면 UI 재사용 금지(캡쳐 금지). 템플릿은 별도 파일/컴포넌트로 분리합니다.
- A4 세로 고정(210mm x 297mm)

B. 첨부 PDF는 “프리뷰가 아니라 원본 페이지 그대로” 포함
- 보험금 청구서 등 첨부파일이 PDF일 경우:
  해당 PDF를 이미지로 변환해서 넣지 말고,
  원본 PDF의 모든 페이지를 final.pdf 뒤에 그대로 append(merge)하세요.
- 이 단계는 pdf-lib로 구현합니다.

권장 파이프라인:
1) Puppeteer로 report_part.pdf 생성 (표지/출동/도면/증빙 이미지/견적/기타 등 템플릿 기반)
2) pdf-lib로 report_part.pdf 로드
3) selectedFileIds 중 type=pdf 파일들을 순서대로 로드 → 모든 페이지 copyPages → addPage로 append
4) 최종 final.pdf 저장/다운로드 응답

--------------------------------------------
4) 각 페이지 템플릿 요구사항
--------------------------------------------

(1) 표지: “현장출동확인서”
- 제목: 현장출동확인서
- 수신/발신
- 사고 정보 테이블: 사고접수번호, 피보험자명, 조사자, 협력업체 (+ 필요 시 주소/출동일시)
- 확인 문구 1줄
- 작성일자(하단 중앙)
- 푸터 문구 1줄
- 배경 이미지 옵션: 넣어도 되지만 텍스트 가독성 확보(흰색 오버레이/반투명 박스)

(2) 출동보고서(1p): 표 기반 정형 문서
- 출동 기본 정보 / 피해자 정보 / 사고 정보 / 복구 내용 / 현장 특이사항(VOC)
- 서명란은 필요 없음(제거)
- 표 스타일: 회색 헤더 + 실선 테두리(보험 문서 느낌)

(3) 도면 페이지(2p): “2. 현장 피해상황 도면”
- 상단: 접수번호/고객사/주소 등 헤더 바(회색 느낌)
- 본문 중앙: 도면 “결과 이미지”만 삽입
  (도면 작성 화면/그리드/컨트롤 UI는 절대 포함 금지)
- img는 object-fit: contain 으로 잘림 없이 중앙 정렬

(4) 증빙자료(3p~): 자동 페이징
- 3페이지 안에 다 안 들어가도 됨. 다음 페이지로 넘어가도 됨.
- 카테고리/탭 구분(현장사진/기본자료/증빙자료/청구자료 등) 제목을 섹션 헤더로 표시
- 사용자가 selectedFileIds로 선택한 파일만 포함
- 이미지 파일: “크게” 보이도록 1개/페이지(또는 2열 옵션)로 렌더
  - 기본은 1개/페이지 권장
  - object-fit: contain(문서/스캔본) 또는 cover(현장사진) 중 상황에 맞게 적용
- PDF 파일: 증빙자료 페이지에 프리뷰로 끝내지 말고, 원본 PDF 페이지 전체를 final.pdf에 append

(5) 견적서: 캡쳐 UI 제거, 2번째 이미지처럼 “정형 표 문서”
- 복구면적/노무비/자재비/합계 등 섹션을 표(table)로 구성
- 페이지가 넘어가도 “섹션 단위로만” 넘어가게 해야 함
  - section에 page-break-inside: avoid; break-inside: avoid 적용
- 표 스타일: 회색 헤더, 실선 테두리, 숫자 정렬 중심
- 웹 UI 버튼(다운로드/미리보기/어떤 아이콘) 절대 포함 금지

--------------------------------------------
5) 출력 품질/접근 이슈 처리
--------------------------------------------
- Puppeteer가 첨부 이미지/PDF URL에 접근 가능해야 합니다.
  접근이 어려우면 서버에서 파일을 읽어 base64(data URL)로 인라인하거나
  임시 signed URL을 발급해 렌더링 시점에 접근 가능하도록 처리하세요.
- 첨부 PDF가 암호화되어 있으면 merge가 실패할 수 있으니 예외 처리/에러 메시지 추가

--------------------------------------------
6) 구현 산출물(필수)
--------------------------------------------
- /pdf-templates/cover.html
- /pdf-templates/field-report.html
- /pdf-templates/drawing.html
- /pdf-templates/evidence-images.html  (이미지용 페이지 생성)
- /pdf-templates/estimate.html
- (선택) /pdf-templates/etc.html
- 서버 엔드포인트:
  POST /api/pdf/download  (payload 입력 → final.pdf 반환)
- pdf-lib로 병합 로직(첨부 PDF append 포함)
- 선택 로직에 따라 섹션/파일 포함 여부 정확히 반영

이 요구사항대로 구현해 주세요.
