누수관리 시스템 – 복구면적/노무비/자재비 연동 규칙 정리
1. 시스템 구성 개요
관리자 페이지

관리자는 엑셀 업로드로 DB를 관리한다.

일위대가 DB

노무비 DB

자재비 DB

이 DB들은 협력사 화면의 견적서 작성에서 데이터 소스로 사용된다.

2. 협력사 화면 구조

견적서 작성 화면에는 4개 탭이 있다:

복구면적 산출표

노무비

자재비

견적서

핵심 원리

복구면적 산출표가 “기준(master)”이다.
여기서 선택된 항목이 노무비와 자재비에 연동된다.

3. 연동 방식 (복구면적 → 노무비/자재비)
3-1. 노무비 연동

복구면적 산출표에서 (공종, 공사명) 선택 시:

노무비는 일위대가 DB에서 조회한다.

조회 조건:

WHERE 공종 = 선택된 공종
  AND 공사명 = 선택된 공사명


해당 조건에 일치하는 모든 노임항목을 자동으로 생성한다.

예:

복구면적: 목공사 – 반자틀
→ 노무비:
   목공사 – 반자틀 – 건축목공
   목공사 – 반자틀 – 보통인부

3-2. 추가 규칙 (철거공사)

특정 공사명의 경우,
노무비에서 철거공사도 함께 따라온다.

해당 공사명:

반자틀

석고보드

도배

마루

생성 로직:

(1) 일위대가DB: (공종=기본공종 AND 공사명=선택공사명)
(2) 일위대가DB: (공종=철거공사 AND 공사명=선택공사명)
→ 둘 다 생성


예:

복구면적: 수장공사 – 마루
→ 노무비 생성:
   수장공사 – 마루 – 내장공
   수장공사 – 마루 – 보통인부
   철거공사 – 마루 – 보통인부


중요:
이 연동된 노무비 행은 수정 불가이다.

3-3. 자재비 연동

자재비는 자재비 DB에서 가져온다.

조회 조건:

WHERE 공사명 = 선택된 공사명


자재비는 공사명에 종속되므로,
공종은 보여주기용이고, 실제 필터는 공사명.

자재비 항목들은 택 1로 선택된다:

공사명 = 마루 → 원목 / 강마루 / 강화마루 중 선택

선택 시 단위/단가 자동 세팅.

이 연동된 자재비 행도 수정 불가이다.

4. 생성/수정/삭제 영향
4-1. 복구면적 산출표

생성/수정/삭제 → 노무비/자재비 자동 반영

상위(master)

4-2. 노무비/자재비 탭에서 개별 추가한 행

복구면적에는 영향을 주지 않는다

독립행

수정 가능

요약:

출발	영향	수정
복구면적	노무비/자재비 영향	불가
노무비/자재비 개별추가	복구면적 영향 없음	가능
5. 노무비 개별 추가 규칙

노무비 탭에서 공종추가 버튼으로 새 행 생성 시:

사용 DB: 노무비 DB

조회 조건: 공종

공사명은 화면에 입력되지만,
노임항목은 공종에만 종속된다.

예:

공종추가: 수장공사, 공사명: 마루
→ 노임항목 리스트:
   - 도배공
   - 내장공
   - 보통인부


이 리스트 중 택 1을 선택해서 행을 생성한다.

이 행은 수정 가능이며
자재비에는 자동 생성되지 않는다.

6. 자재비 개별 추가 규칙

자재비 탭에서 수동으로 추가할 수도 있다.

사용 DB: 자재비 DB

조회 조건: 공사명

예:

공사명: 마루
→ 자재항목 리스트:
   - 원목
   - 강마루
   - 강화마루
   (택 1)


선택하면 단위/단가 자동 세팅.

노무비와는 연동 없음.

7. 손해방지 / 피해세대 공종 노출 규칙

관리자가 누수건을 접수할 때:

손해방지 선택 → 원인세대 공종만 노출

피해세대 복구 선택 → 피해세대 공종만 노출

이 필터는 공종DB 시트 기준.

다른 조건 없음.

8. 수정 가능 여부 요약
구분	생성 위치	출발	수정
연동행	노무비/자재비	복구면적	수정 불가
독립행	노무비/자재비	공종추가 or 자재추가	수정 가능
9. DB 사용 정리
상황	참조 DB	조회 기준	선택
복구면적 → 노무비	일위대가 DB	공종 + 공사명 (+철거공사 규칙)	모두
복구면적 → 자재비	자재비 DB	공사명	택 1
노무비 개별 추가	노무비 DB	공종	택 1
자재비 개별 추가	자재비 DB	공사명	택 1
10. 핵심 흐름 요약 (1페이지 버전)

복구면적 산출표가 모든 출발점

노무비는 일위대가 DB

공종+공사명 → 일치하는 모든 노임항목 생성

특정 공사명은 철거공사도 추가 (반자틀/석고보드/도배/마루)

자재비는 자재비 DB

공사명 → 자재항목 택1

연동된 행은 수정 불가

노무비/자재비 직접 추가는 수정 가능

직접 추가는 서로 연동되지 않음

11. 개발자가 반드시 이해해야 하는 관점

“복구면적 연동행”과

“노무비/자재비 독립행”을 구분해야 한다.

이를 위해 각 행에 flag가 필요하다:

isLinkedFromRecovery   (true/false)


true → 수정불가

false → 수정가능

12. 예시 시나리오

복구면적: 수장공사 – 마루

→ 노무비 생성:

수장공사 – 마루 – 내장공

수장공사 – 마루 – 보통인부

철거공사 – 마루 – 보통인부

→ 자재비 생성:

수장공사 – 마루 (행 생성)

자재항목: 원목/강마루/강화마루 중 택1

단위/단가 자동

→ 필요하면 추가 가능:

노무비에서 수장공사 추가 → 노임항목 택1
자재비 자동 생성 없음

13. 목표

Replit 개발자가 아래를 구현할 수 있어야 한다:

복구면적에서 자동 생성 + 수정불가

노무비/자재비에서 독립 추가 + 수정가능

DB 조회 키 정확히 반영

철거공사 자동 생성 로직 포함

이게 전체 규칙 완성 요약본이다.
이대로 구현하면 모든 케이스가 처리된다.
