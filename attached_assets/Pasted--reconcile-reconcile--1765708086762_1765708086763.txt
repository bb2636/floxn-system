말씀하신 reconcile 로직 추가에도 불구하고, 복구면적 산출표에서 “수정”하거나 “삭제(-)”했을 때 자재비가 여전히 변경되지 않습니다.
즉, 현재 문제는 reconcile 로직의 존재 여부가 아니라, “언제/어떤 값으로 호출되는지”가 잘못된 상태입니다.

아래를 순서대로 반드시 확인 및 수정해 주세요.

1️⃣ 가장 중요한 확인: syncMaterialFromRecoveryArea가 실제로 호출되는가?
즉시 추가해서 확인할 로그
console.log("[SYNC CALL]", {
  time: Date.now(),
  recoveryCount: recoveryAreaRows.length,
  recoverySnapshot: recoveryAreaRows.map(r => ({
    id: r.id,

    area: r.recoveryArea
  }))
});


👉 복구면적 산출표에서

공사명 변경

면적 변경

행 삭제(-)

할 때마다 이 로그가 찍혀야 합니다.

❗ 만약 안 찍히면:

reconcile 로직은 “있지만”

복구면적 변경에 의해 호출되지 않고 있음 → 구조적 버그

2️⃣ useEffect dependency 문제 (거의 확정)

현재 증상은 아래 케이스와 100% 일치합니다.

복구면적 행 내부 필드(workName, area)만 변경

recoveryAreaRows 배열 참조(reference) 는 그대로

useEffect dependency가 [recoveryAreaRows] 만 있음
→ React는 “변경 없음”으로 판단 → sync 미실행

필수 수정

아래 중 하나로 고쳐야 합니다.

방법 A (가장 안전)
useEffect(() => {
  syncMaterialFromRecoveryArea();
}, [JSON.stringify(recoveryAreaRows)]);

방법 B (권장, 성능 고려)
const recoverySignature = useMemo(
  () => recoveryAreaRows.map(r =>
    `${r.id}|${r.workName}|${r.recoveryArea}`
  ).join(','),
  [recoveryAreaRows]
);

useEffect(() => {
  syncMaterialFromRecoveryArea();
}, [recoverySignature]);


👉 이거 안 하면
“로직은 있는데 수정/삭제에 반응 안 하는 상태”가 계속됩니다.

3️⃣ stale 삭제 로직이 “실제 autoRow”에 적용되는지 검증

[MATERIAL RECONCILE] 로그가 찍힌다고 해서 정상 동작은 아닙니다.
아래 로그를 reconcile 직후에 반드시 찍어 주세요.

console.log("[RECONCILE RESULT]", {
  nextKeys: Array.from(nextAutoMap.keys()),
  finalRows: materialRows.map(r => ({
    id: r.id,
    auto: r.isAutoGenerated,
    key: r.autoKey,
    work: r.workName
  }))
});


👉 제가 보고 싶은 것:

복구면적에서 걸레받이를 지웠다면
→ nextKeys에 걸레받이 key ❌
→ finalRows에도 걸레받이 ❌

지금은 nextKeys는 바뀌는데 finalRows는 그대로일 가능성이 큽니다.

4️⃣ 또 하나의 핵심: 과거 행이 autoRow로 분류되지 않는 문제

이전 단계에서 생성된 자재비 행 중 일부가:

isAutoGenerated = false

또는 isOverridden = true

로 남아 있다면, reconcile 삭제 대상에서 빠집니다.

필수 정책 정정

자동연동 대상 공사명(합판/석고보드/걸레받이 등)은

생성/갱신 시 무조건 isAutoGenerated=true

복구면적에서 사라지면 overridden 여부와 관계없이 제거

❌ “사용자가 수정했으니 남긴다” 정책 금지
→ 지금 문제의 원인이 됨

5️⃣ 지금 단계의 결론 (중요)

“reconcile이 수정/삭제 시점에 실행되지 않거나, 실행돼도 입력이 갱신되지 않는 문제”입니다.

그래서:

노무비는 잘 따라가고

자재비만 과거 상태가 남아 있는 겁니다.

6️⃣ 내가 요구하는 결과 (다시 명확히)
테스트
