현재 노무비는 복구면적 산출표 변경(추가/수정/삭제)을 잘 따라가는데, 자재비는 과거에 생성된 자동연동 항목이 남아있고(복구면적에 없는 항목), 복구면적에서 수정한 내용도 반영이 안 됩니다.
즉, 자재비 sync가 append/update만 하고, stale auto rows 제거/대체(reconcile)가 불완전합니다.

1) 기대 동작(요구사항)

자재비의 자동연동 대상(합판/석고/석고보드/몰딩/걸레받이/도배/마루/장판)은

복구면적 산출표가 바뀔 때마다 항상 현재 복구면적 상태 기준으로 재계산

복구면적에 더 이상 존재하지 않는 항목은 자재비에서 자동으로 제거

복구면적에서 항목 변경(예: 걸레받이 → 석고보드) 시

자재비도 걸레받이는 사라지고

석고보드는 생기거나 수량 갱신되어야 함

2) 지금 발생하는 현상(반증)

복구면적 산출표에 없는 자재(예전에 추가했던 걸레받이 등)가 자재비에 남아있음

복구면적에서 공사명/면적 수정해도 자재비가 갱신되지 않는 느낌

3) 원인 후보 (우선순위대로 체크)
A) 자재비에서 “자동연동 행”으로 분류가 안 되어 삭제 대상에서 빠짐

과거에 만들어진 행이 isAutoGenerated=false 이거나

isOverridden=true로 찍혀서 “수동행”으로 취급되어 계속 남아있을 가능성이 큼

✅ 조치:

자동연동 대상 workName이면, 최초 생성/갱신 시 무조건 isAutoGenerated=true 유지

isOverridden는 “값 덮어쓰기 방지” 용도일 뿐이고,
복구면적에서 항목이 사라졌으면(is no longer present) 제거 정책을 적용해야 함

정책 확정 필요:

복구면적에서 항목이 완전히 사라졌는데도 isOverridden이면 남길지?
제 요구는: 복구면적 기반 자동연동 항목은 원본에서 사라지면 자재비에서도 제거 (수정해둔 값이 있더라도)

B) autoKey가 변경되어 “같은 항목”으로 매칭이 안 됨

autoKey에 자재항목(materialItem)이 들어가는데,
선택 값이 ""/undefined/선택/실제항목으로 흔들리면
동일 항목도 다른 key로 인식되어 갱신/삭제가 꼬입니다.

✅ 조치:

자동연동 항목에 한해 autoKey를 단순화:

공종|공사명까지만 사용하거나

materialItem이 없으면 항상 "__NONE__"로 고정

const norm = (v:any) => (v ?? "").toString().trim();
const itemKey = norm(materialItem) || "__NONE__";
const autoKey = `${norm(trade)}|${norm(workName)}|${itemKey}`;

C) syncMaterialFromRecoveryArea가 “삭제(reconcile)”를 실제로 안 하고 있음

nextAutoRows는 계산하지만

기존 autoRows 중 nextAutoKeys에 없는 것을 제거하지 않으면
예전 항목이 계속 남습니다.

✅ 조치(필수): 아래 로직대로 구현

4) 필수 구현: 자재비 reconcile (삭제/대체 포함)

syncMaterialFromRecoveryArea()는 아래 순서로만 동작해야 합니다.

복구면적 rows 기준으로 nextAutoMap 생성 (key=autoKey, value=계산 결과)

기존 materialRows에서

manualRows: 자동연동 대상이 아닌 수동행 유지

autoRows: isAutoGenerated=true인 행들만 추출

삭제 단계:

기존 autoRows 중 autoKey가 nextAutoMap에 없는 행은 제거

업데이트/생성 단계:

autoKey가 존재하면 기존 row 재사용하며 수량/단위 업데이트

없으면 신규 생성

최종 replace:

setMaterialRows([...manualRows, ...reconciledAutoRows])

✅ 제가 확인하고 싶은 건 딱 2개입니다.

그런데도 자재비에 걸레받이 row가 남아있으면 그 row가 왜 “삭제 대상이 아닌지”(auto/overridden/key 불일치) 로그로 바로 드러납니다.

6) 확인 시나리오(정답 기준)

복구면적에서 걸레받이 추가 → 자재비에 걸레받이 생김

같은 행을 석고보드로 변경
✅ 자재비에서 걸레받이는 사라지고 석고보드로 바뀌거나 새로 생성되어야 함

복구면적에서 해당 행 삭제
✅ 자재비에서도 관련 자동연동 행이 제거되어야 함
