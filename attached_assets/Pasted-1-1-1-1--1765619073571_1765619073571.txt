1) 복구면적산출표 → 노무비는 정상 연동, 자재비만 이상함
기대 동작

복구면적산출표에서 “합판” 1개 추가
→ 노무비 1개 연동
→ 자재비도 합판 자동연동 행 1개만 생성되어야 함
(추가로 빈 행/수동행이 자동 생성되면 안 됨)

실제 동작

합판 1개 추가 시 자재비에

연동된 자동 항목 1개는 생김 (OK)

그런데 독립적으로 추가된 빈 행이 2개가 추가로 생김 (문제)

즉, 자재비에서 “자동 연동 행 + 빈 수동 행 2개”가 동시에 생성되고 있습니다.

원인 후보 (우선순위)
A) 자재비 컴포넌트가 기본적으로 빈 행 2개를 자동 생성

material-cost-section.tsx 쪽에서

초기 렌더 시 if (rows.length===0) addRow() 같은 로직이 2회 실행되거나

StrictMode/dev effect 중복, 또는 dependency 변화로 effect가 2번 실행되어

“빈 수동 행”이 2개 생기는 패턴이 흔합니다.

✅ 확인 요청:

자재비 화면에서 행이 생성되는 코드는 아래 중 어디인지 찾고,

addRow(), createEmptyRow(), initRows() 같은 함수 호출이 자동으로 2번 발생하는지 확인해주세요.

B) syncMaterialFromRecoveryArea에서 “수동행 보존” 로직이 오히려 수동행을 생성

isOverridden 보존/병합 과정에서

빈 행을 manualRows로 분류해서 계속 유지하거나,

“최소 2행 유지” 같은 UI 정책이 섞여 있을 수 있습니다.

최종 setState가 prev + autoRows 형태면,

빈 수동행이 있든 없든 계속 누적됩니다.

요구사항 (명확히)

자동 연동으로 인해 자재비에 빈 수동행이 자동 생성되면 안 됨

“빈 행 제공”이 UX 정책이라면:

사용자가 + 행추가 눌렀을 때만 생성

또는 “초기 1회만” 생성하되 sync로 인해 다시 생기지 않도록 방어

2) 배포버전에서 관리자에서 추가한 “장소”가 화면에 안 뜸 (dev에서는 정상)
증상

개발버전(dev)에서는 관리자에서 추가한 “장소”가 바로 조회/표시됨

배포버전(prod)에서는 같은 기능이 조회 목록에 안 뜸

원인 후보 (우선순위)

prod가 dev와 다른 DB를 보고 있음

DATABASE_URL / schema / project env가 다름

prod에 마이그레이션이 안 되어 테이블/컬럼이 다르거나

seed 데이터/인덱스/조건이 불일치

조회 API가 필터 조건 때문에 0건으로 나오고 있음

예: companyId, isActive, deletedAt is null, tenantId 같은 조건이 prod 데이터와 안 맞음

권한(401/403) 또는 쿠키/토큰 전달 문제

prod에서만 auth가 깨져서 실제로는 API가 실패하거나 빈 배열로 응답
